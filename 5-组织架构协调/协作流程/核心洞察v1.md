# 核心资料提炼（第一版）

## 一、页面承接的核心问题

### 1. 用户在最前面就被挡住了
- 年龄确认弹窗是所有新用户必经步骤。
- 当前分析口径里，只有完成年龄确认并停留超过 5 秒的用户才算进入承接分析。
- 3/23 总会话 1423，但进入该分析对象的只有 92，说明前置流失非常重。

**判断：** 现在不是单纯“页面内容差”，而是用户在进入真实内容前就已经大规模流失。

### 2. 页面承接目标需要更明确
- 现有页面设计思路是：内容制造欲望 → 测试锁定需求 → 产品承接体验 → App 实现复购。
- 但当前实际业务目标已经收敛为：让游客到达产品页后完成支付，拿到第一笔订单。

**判断：** 现阶段应优先服务“首单支付”，不能让过多内容路径分散主目标。

### 3. 转化路径里存在硬摩擦点
根据《creviatech.com 转化路径优化：渐进式迭代计划》：
- 广告落地 URL 可能错误，存在 `/product`、`/products`、`/theater` 返回 404 的问题。
- 结账页地址填写过重，字段多，且缺少自动填充。
- 结账页支付信任信号不足，欧美用户在付款前看不到足够明确的支付方式背书。

**判断：** 这三项里，URL 和结账信任问题优先级最高，属于先修“漏水点”。

### 4. 当前页面结构想法很多，但主路径还不够聚焦
页面设计文档里规划了：首页、社区、剧场、女性专栏、测试、商品、App、订阅等多个模块。

**判断：** 长期看这是一套内容生态，但现阶段会带来路径分散。第一阶段应优先强化：
- 广告 → 产品页 → 结账
- 广告 → 剧场页 → 产品页 → 结账

## 二、漏斗分析的关键结论

### 1. 大盘问题非常明确：大量用户秒跳
- 3/22 总 Sessions：1371
- 停留时长中位数：3 秒

**结论：** 半数以上用户在 3 秒内流失，说明当前的首屏吸引力、页面匹配度、访问前预期管理至少有一项明显偏弱。

### 2. 目前最关键不是“买更多流量”，而是提升 Baseline 之后的留存质量
HVU 三层定义：
- Baseline：PV ≥ 2 且时长 > 3s
- Engaged：满足 Baseline 且时长 > 60s
- High-Intent：满足 Engaged 且访问 checkout/cart/shipping/scenarios/products 页面

**结论：** 你现在要做的不是只盯 CTR，而是把流量从“基础存活”推进到“深度沉浸”和“高意向”。

### 3. Facebook 仍然是核心增长引擎，但成本高
- Facebook：$224.64 花费，286 Baseline，40 Engaged，18 High-Intent
- Cost Per Baseline：$0.79
- Cost Per Engaged：$5.62
- Cost Per High-Intent：$12.48

**结论：** Facebook 能持续给出相对稳定的漏斗，但前端流量成本偏高。短期不应放弃，而应继续作为主阵地优化素材和承接。

### 4. Exoclick 纯度高，但内容留存差
- Exoclick：$9.40 花费，59 Baseline，4 Engaged，4 High-Intent
- Cost Per Baseline：$0.16
- Cost Per Engaged / High-Intent：$2.35

**结论：** Exoclick 的问题不在买量，而在内容承接。说明某些渠道拿到用户后，站内内容还不够把人继续留下来。

### 5. 剧情详情页路径明显优于直接产品详情页路径
从素材榜看：
- 跳剧场详情的素材里，有多个进入 High-Intent
- 跳产品详情的素材，多数有 Baseline/Engaged，但 High-Intent 为 0

**结论：** 当前阶段，剧情内容页比直接产品页更容易把用户推到高意向。说明“内容先吸住，再导商品”这条路径更值得优先强化。

## 三、下一阶段最该做的 3 个渐进动作

### 动作 1：先修路径硬错误和结账信任问题
负责人：技术 + 你确认优先级

优先项：
1. 全量核查广告落地 URL，杜绝 404
2. 在结账页补足支付方式图标和信任提示
3. 尽可能减少地址填写阻力（先做最小可行改动）

**原因：** 这些属于“修漏点”，对转化影响直接，且改动小。

### 动作 2：把剧情页作为主承接验证方向，不急着强推直达产品页
负责人：投手 + 你

执行方式：
1. 继续重点测试“视频/剧情 → 剧场详情页”方向
2. 对剧场页里的商品卡、Order now、关联商品做更清晰观察
3. 产品详情页流量先保留，但不要作为唯一主路径

**原因：** 当前数据已经表明剧情页更有机会把用户推进到 High-Intent。

### 动作 3：围绕“把普通用户变成沉浸用户”做页面小步优化
负责人：技术 + 你

优先关注：
1. 首屏 3 秒内是否说清楚价值
2. 年龄确认后的首屏吸引力是否足够强
3. 页面内是否有明确继续浏览和跳转路径
4. 是否有内容模块能把用户从浏览推进到剧情/商品

**原因：** 现在大量流量倒在 3 秒内，先把秒跳压下来，比加更多功能更重要。

## 四、现阶段顾问判断

### 当前最优先的顺序
1. 修硬错误
2. 优化剧情承接页
3. 降低首屏秒跳
4. 再看产品页转化细节

### 当前不建议优先做的事
- 大规模重构整站
- 一次性加很多新功能
- 让 6 个投手同时跑太多不同路径
- 过早做重组织协同

### 当前一句话策略
**先把“能进站的人留下来，并顺利走到高意向页”这件事跑顺，再谈放量。**
