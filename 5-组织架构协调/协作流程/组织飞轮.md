# Crave AI 组织飞轮

## 一、飞轮机制

Crave AI 的增长飞轮是一个持续运转的决策-执行-验证循环：

```
数据 → 判断 → 方向 → 执行 → 结果 → 数据（新一轮）
```

飞轮一旦转起来，每一圈都会比上一圈更快、更准。飞轮的驱动力只有一个：**转化提升**。

### 飞轮四阶段

| 阶段 | 内容 | 核心问题 |
|------|------|----------|
| **数据** | 看板数据、日报、素材表现、路径漏斗 | 看到了什么？ |
| **判断** | 问题归因：是投放、素材还是承接？ | 什么在拖转化？ |
| **方向** | 给投手/技术下达次日优化方向 | 下一步做什么？ |
| **执行** | 投手测试、技术改版、素材生产 | 做了吗？做得对吗？ |

## 二、角色定位与飞轮贡献

### 统筹（你）

- 飞轮核心节点：看数据 → 做判断 → 下方向
- 每天固定动作：早盘数据判断 + 晚复盘
- 每周固定动作：风险预警 + 次周优先级排布
- 质量要求：判断要快、方向要准、不同时改太多变量

- 当前状态：响应可跟上，排期是主要风险点

## 三、当前飞轮节奏

### 每日节奏

| 时间 | 动作 | 谁做 |
|------|------|------|
| 早晨 | 查看昨日数据，判断主要问题 | 统筹 |
| 早晨 | 下达当日优化方向 | 统筹→投手+技术 |
| 全天 | 投手执行测试，投手+技术执行优化 | 投手+技术 |
| 傍晚 | 投手日报提交 | 投手 |
| 傍晚 | 日报汇总，判断是否调整方向 | 统筹 |

### 每周节奏

| 时间 | 动作 | 谁做 |
|------|------|------|
| 周初 | 本周目标确认，风险预警 | 统筹 |
| 周中 | 一次全员培训（节奏型）| 统筹组织 |
| 周五 | 周复盘，识别下周一优先项 | 统筹 |
| 随时 | 风险预警（提前一周）| 统筹 |

## 四、当前阶段

### 阶段名称
第一阶段：问题识别与基础优化

### 阶段目标
1. 分清问题来自投放、素材还是页面承接
2. 建立统一日报和复盘机制
3. 找到可以继续验证的素材方向
4. 找到最需要优先优化的核心落地页

### 阶段完成标志
- 能稳定区分投放问题/素材问题/页面问题
- 有 1–2 个相对稳定的素材方向
- 有 1 个明确进入迭代的核心页面
- 日常复盘机制运转稳定

### 当前最优先顺序
1. 修路径硬错误（404、结账信任）
2. 优化剧情承接页（数据已证明剧情路径 > 直接产品页）
3. 降低首屏秒跳
4. 再看产品页转化细节

### 不做的清单
- 不做复杂人员协同管理
- 不做大规模网站重构
- 不做大范围预算放量
- 不同时改太多变量

## 五、核心指标体系

### 结果指标
| 指标 | 定义 | 关注点 |
|------|------|--------|
| 高质量用户数量 | 进入高意向跳转页的用户（P90停留 + 访问checkout/cart/products）| 每周趋势 |
| 高质量用户成本 | 花费 / 高质量用户数 | 持续降低 |
| 转化率 | High-Intent / 总 Sessions | 持续提升 |
| ROI | 收入 / 花费 | 稳定 |

### 漏斗分层（用户质量分层）
| 层级 | 定义 |
|------|------|
| Baseline | PV ≥ 2 且停留 > 3秒 |
| Engaged | Baseline + 停留 > 60秒 |
| High-Intent | Engaged + 访问 checkout/cart/shipping/scenarios/products |

### 渠道现状（当前数据）
| 渠道 | 特点 |
|------|------|
| Facebook | 主力引擎，成本高但漏斗稳定 |
| Exoclick | 用户纯度高，但站内留存差，承接有问题 |

## 六、风险预警

### 每周必看风险
1. **FB 养号风险** — 账号频繁被封影响投放连续性
2. **素材操作风险** — 大量素材跑起来后操作管理跟不上
3. **页面排期风险** — 技术修改排期跟不上投放优化节奏
4. **团队效率风险** — 员工不知道如何干，效率低

### 提前预警要求
- 提前一周预警未来一周风险
- 方便统筹并行处理，不堆积
