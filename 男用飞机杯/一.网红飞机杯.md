# 网红飞机杯 · 创作者落地页设计方案

## 一、网页设计方案

### 目标定位

这个页面的核心目标：让不同类型的网红一眼看懂"合作对我有什么好处"，并完成留资/报名动作。

---

## 整体结构（7个模块，线性单页）

```
┌────────────────────────────────────┐
│  首屏 Hero                          │
│  一句话价值 + 核心钩子 + CTA          │
└────────────────────────────────────┘
         ↓
┌────────────────────────────────────┐
│  收益展示模块                        │
│  佣金比例 + 已结算数据（社会证明）     │
└────────────────────────────────────┘
         ↓
┌────────────────────────────────────┐
│  门槛说明模块（消除顾虑）             │
│  粉丝量/平台/地区 → 都能做            │
└────────────────────────────────────┘
         ↓
┌────────────────────────────────────┐
│  合作流程模块                        │
│  3步极简流程：注册→选品→出单          │
└────────────────────────────────────┘
         ↓
┌────────────────────────────────────┐
│  达人分层权益表                      │
│  微V/小V/中V/大V 对应不同权益         │
└────────────────────────────────────┘
         ↓
┌────────────────────────────────────┐
│  信任背书模块                        │
│  支付方式图标 + 合规标志 + 实际案例    │
└────────────────────────────────────┘
         ↓
┌────────────────────────────────────┐
│  底部 CTA + 留资表单                 │
│  平台/地区/粉丝量 3个字段 + 提交       │
└────────────────────────────────────┘
```

---

## 各模块设计重点

### 首屏 Hero（决定3秒留存）

- **主标题**：直给利益，不说产品，说赚钱
  - 英文示例：Turn your following into $1,200+/month
  - 中文：你的粉丝，每月可以多赚一笔
- **副标题**：消除门槛疑虑 → 10K followers is enough to start
- **CTA 按钮**：唯一主按钮，颜色对比度高，文案用动词开头
  - `Apply in 30 seconds →`
- **背景**：沿用暗色系 + 粉紫渐变，保持品牌一致性

### 收益展示模块

- 展示佣金区间：**40% – 75%**（对应地区/粉丝量）
- 强化社会证明：已向创作者结算 $XXX,XXX
- 关键数据用大字号突出，不要段落文字

### 门槛说明模块（5维标签的最大价值在这里）

- 按平台分列：OnlyFans / Fansly / TikTok / Instagram
- 每个平台用1句话说核心优势，不展开
- 底部加一行：其他平台也可咨询 → 降低排除感

### 达人分层权益表

| 层级 | 粉丝量 | 佣金 | 专属支持 |
|------|--------|------|----------|
| 新人 | <10K | 40% | 素材扶持 |
| 成长 | 10K–50K | 50% | 案例指导 |
| 专业 | 50K–200K | 60% | 专属对接 |
| 头部 | 200K+ | 75% | 定制方案 |

### 信任模块（Crave AI 当前缺失的关键项）

- 支付方式图标：PayPal / Stripe / Wire Transfer
- 合规徽章：CCPA / GDPR
- 提现周期：**每周结算，T+3 到账**

### 留资表单（极简）

- 只问3个字段：平台 / 地区 / 粉丝量
- 不问姓名/邮箱（可二步收集），先降低心理成本
- 提交后：直接显示"下一步由专属对接人联系你"

## 视觉规范

| 元素 | 建议 |
|------|------|
| 背景 | `#0D0D0D` 黑底 |
| 主色 | `#FF5C8D` 粉色 / `#9B5CF7` 紫色渐变 |
| 文字 | `#FFFFFF` 主文 / `#AAAAAA` 次文 |
| CTA 按钮 | 粉紫渐变，圆角8px，阴影增强点击感 |
| 字体 | 英文 Inter/DM Sans，中文苹方 |
| 移动端 | 首屏 CTA 必须在 viewport 内可见，不需要滑动 |

## 二、基于 Crave AI 历史问题的风险提示

### 风险 1：年龄验证弹窗大量截流

**实际数据**：1423 会话中，只有 92 人通过验证进入承接分析（通过率约 6%）

**该页面对应风险**：如果这个达人招募页也加了年龄验证，会直接截掉大量潜在合作者

**建议**：
- 达人招募页与产品页分开部署，招募页不加年龄弹窗
- 或弹窗改为一键确认（不要让用户输入生日）

---

### 风险 2：首屏 3 秒内秒跳

**实际数据**：Crave AI 主站停留中位数 3 秒，超过半数用户在 3 秒内离开

**该页面对应风险**：首屏如果放的是产品介绍而不是利益点，网红用户会立刻关掉

**建议**：
- Hero 区禁止放产品图，第一句话必须是钱
- 首屏不超过 3 个信息点
- 移动端首屏禁止有横向滑动或视频自动播放

---

### 风险 3：落地 URL 出现 404

**已发生**：/product、/products、/theater 路径有返回 404 的记录

**该页面对应风险**：广告投放指向的 URL 若变动或拼写错误，直接丢失所有付费流量

**建议**：
- 上线前全量核查所有跳转链接
- 用 UTM 参数追踪每个流量入口
- 404 页面必须有兜底跳转，不能是空白

---

### 风险 4：结账/留资信任感不足

**实际问题**：Crave AI 结账页缺少支付方式图标和信任标志，欧美用户放弃支付

**该页面对应风险**：网红填写留资表单前，如果看不到支付方式和结算证明，会认为是诈骗

**建议**：
- 表单上方必须有真实结算案例（截图或数据）
- 支付方式 logo 必须在表单附近可见
- 加上隐私承诺一行文字（信息不会被分享给第三方）

---

### 风险 5：5 维标签没有转化为页面个性化

**当前状况**：XLS 里已有详细的平台/地区/年龄/粉丝量/语言的话术差异，但页面是一套通用版本

**风险**：北美大V 和东南亚新手对"好处"的理解完全不同，一套话术两边都说服不了

**建议**：
- 短期：页面顶部加地区/平台切换 tab，切换后显示对应话术
- 中期：根据来源 UTM 自动匹配话术版本（技术改动小，影响大）
- 不建议现在做 AI 动态个性化，先用静态多版本验证哪个方向有效

---

## 风险优先级汇总

| 优先级 | 风险项 | 难度 | 影响 |
|--------|--------|------|------|
| P0 | 首屏3秒内容是否命中利益点 | 低 | 极高 |
| P0 | 年龄验证与招募页分离 | 低 | 极高 |
| P1 | URL 404 全量核查 | 低 | 高 |
| P1 | 结算信任模块补足 | 中 | 高 |
| P2 | 5维话术多版本页面 | 中 | 中高 |
